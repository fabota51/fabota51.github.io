Lagerdetect

Python YOLOv11 Raspberry Pi Computervision 2025

Lagerdetect ist ein Projekt, das ich 2025 wärhrend eins Praktikums als Erweiterung des Bestehenden LagerPi gestartet habe. Ziel war es, das Bestehende system wieder zum laufen zu bringen/verbessern. Das System besteht aus einem Raspberry Pi mit angeschlossener Kamera, der regelmäßig Fotos von den Getränkekisten macht und Flaschen auf den Bildern erkennt. Anschließend, zählt er sie und zeigt die Ergebnisse auf einer lokalen Website im Netzwerk.

01 Bestandsaufnahme

Der ursprüngliche Raspberry Pi 4 ließ sich nicht mehr starten – keine SSH-Verbindung, kein Bild am Monitor, keine Reaktion auf verschiedene SD-Karten mit unterschiedlichen Betriebssystemen. Die LEDs zeigten zwar Strom und SD-Erkennung, aber kein Boot. Diagnose: Hardware-Defekt. Lösung: ein funktionierender Raspberry Pi 3B+ wurde bereitgestellt.

02 Setup & Entwicklungsumgebung

Betriebssystem installiert, SSH direkt im Raspberry Pi Imager aktiviert. Auf dem Pi: VS Code Server und Git. Auf der lokalen Maschine: VS Code mit SSH-Extension. Damit war Remote-Entwicklung, Repo klonen und Pushen direkt vom Pi aus möglich – kein Bildschirm am Pi nötig.

03 LagerPi v2 wieder zum Laufen bringen

Das bestehende GitHub-Repo auf den neuen Pi geklont, Pfade angepasst. Website startete, aber ohne Kamerabilder. Ursache: fehlende Einträge in boot/firmware/config.txt. Nach der Korrektur lieferte die Kamera Bilder und die Seite zeigte den aktuellen Lagerbestand.

04 Erkennung verbessern

Das alte System erkannte bei guten Lichtverhältnissen 113 Flaschen, bei Dunkelheit nur 12 – zu unzuverlässig. Entscheidung: Umstieg auf YOLOv11s (small, wegen limitierter Pi-Ressourcen). Da kein passendes vortrainiertes Modell für Flaschenkappen existierte, habe ich auf Roboflow mehrere Datensätze kombiniert. Die Label-Formate der verschiedenen Quellen (unterschiedliche Klassennamen, leere Flaschen, verschiedene Getränkesorten) habe ich mit einem eigenen Skript vereinheitlicht. Ergebnis: ein Megadatensatz mit 2.300 gelabelten Bildern, die ich einmal manuell auf offensichtliche Fehler geprüft habe.

05 Training & Integration

Training auf dem Laptop, parallel dazu Integration ins System. Problem: Die Erkennungsfunktion lief bei jeder HTTP-Anfrage – Ladezeit 36 Sekunden. Lösung: Ergebnisse cachen, Modell einmalig beim ersten GET / laden und im RAM halten. Hinzugefügt: manueller Auslöse-Button und eine zeitgesteuerte automatische Erkennung. Die Tabelle zeigt pro Getränkekiste ein Bild und den Namen. Das alte Übersichtsbild wurde durch ein Diagramm ersetzt, das die Werte der letzten Scans visualisiert.

06 Settings-Seite

Die Settings-Seite wurde in drei Bereiche aufgeteilt: Kisten definieren, Kisten-Einstellungen und Alarm. Eigene API-Endpoints für das Abrufen und Ändern der Kisten-Konfiguration. Alarm-Funktion: bei Unterschreiten eines definierten Flaschenschwellenwerts wird automatisch eine Benachrichtigung ausgelöst. Kistenpositionen können direkt im Bild auf der Settings-Seite per Mausklick markiert und gespeichert werden – kein externes Tool mehr nötig.

07 Fazit

Das System erkennt zuverlässig Flaschen und stellt die Ergebnisse auf einer lokalen Website dar. Neben Computervision und YOLO habe ich Remote-Entwicklung mit SSH vertieft.
