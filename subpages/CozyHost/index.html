CozyHost

PHP Docker MariaDB Netbird Minecraft Server-Hosting Schulprojekt 2026

CozyHost ist ein Schulprojekt – ein selbst gebautes Minecraft-Server-Hosting-Panel aus PHP, Docker und Netbird. Das Projekt entstand unter extremem Zeitdruck: Theoretisch standen die Winterferien plus zwei Wochen danach zur Verfügung – die eigentliche Umsetzung passierte dann in genau diesen letzten zwei Wochen. Das Ergebnis ist ein Proof-of-Concept, welcher mit viel Klebeband zusammenhält.

01 Die Idee

Freunde sollen per Klick einen eigenen Minecraft-Server starten können – ohne Port-Forwarding, ohne statische IP, ohne offene Firewall. Das Panel legt für jeden Server automatisch eine Netbird-Adresse an und regelt den Zugriff darüber: Nur eingeladene Nutzer kommen durch. Im Hintergrund läuft alles in Docker-Containern, die das Panel direkt über die Docker-API spawnt. Das Ganze Projekt kann über einen einziges Docker-Compose-File gestartet werden, welches alle Services (Panel, MariaDB, Minecraft-Server) hochzieht.

02 Wie es funktioniert

Das Backend ist ein PHP-Frontend, das in einem eigenen Docker-Container läuft und mit drei weiteren Services kommuniziert: einer MariaDB-Datenbank für Nutzer- und Serverdaten sowie der lokalen Docker API (über den freigegebenen Socket), um neue Minecraft-Container zu starten, zu stoppen und zu verwalten. Für das Netzwerk spricht das Panel direkt gegen die Netbird REST-API: Pro Server wird eine eigene Netbird-Gruppe angelegt, ein Setup-Key generiert und eine Zugriffsrichtlinie erstellt – so kommt nur der berechtigte Nutzer an seinen Server.

03 Nutzerverwaltung & Profil

Das Panel kennt genau einen Admin-Account, der über die Oberfläche neue Netbird-Nutzer anlegen und deren Zugriff auf einzelne Server steuern kann. Jeder Nutzer existiert dabei als eigene Gruppe im Netbird-Netzwerk – der Admin entscheidet, welche Nutzergruppe welchen Server erreichen darf. Auf der Profilseite lässt sich der persönliche Netbird-API-Token hinterlegen und das Passwort ändern.

04 Aktueller Stand & Einschränkung

CozyHost funktioniert – solange alle Mitspieler den Netbird-Client installiert haben. Über das Overlay-Netzwerk erreichen sie den Server direkt, ohne Port-Forwarding oder offene Firewall. Was aktuell nicht geht: den Server öffentlich zugänglich machen. Netbird-Reverse-Proxies unterstützen bisher nur HTTP-Traffic – einen Minecraft-Server (TCP) darüber nach außen durchzureichen ist damit noch nicht möglich. Wer keinen Netbird-Client hat, kann also noch nicht joinen.

Der Plan: warten, bis Netbird den nötigen Traffic-Support nachrüstet. Dann soll kein komplettes VM-System mehr gebaut werden – stattdessen ist ein Plugin für PelicanPanel geplant, das Netbird direkt auf den Wings installiert, Adressen automatisch vergibt und das Port-Forwarding übernimmt. Sauberer, kleiner, wartbarer.

05 Fazit

CozyHost ist kein fertiges Produkt. Es ist ein Schulprojekt, das in zwei Wochen entstand, die Kernidee beweist und genau so aussieht. Die Architektur zeigt, wohin es gehen soll – und sobald Netbird die Lücke schließt, wird aus dem Klebeband-Konstrukt hoffentlich etwas Vernünftiges.
